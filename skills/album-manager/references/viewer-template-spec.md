# Viewer Template Specification v2.0.0

## Overview

The thumbnail viewer is a self-contained HTML file with embedded base64 images,
generated by the Python pipeline in the album-manager and photo-search skills.

**Canonical templates** now live in `assets/`:
- `assets/viewer-template.html` — The photo viewer with all 4 view modes, 3 themes, gallery overlay, and Cowork Actions Panel
- `assets/index-template.html` — The query dashboard (Option B) listing all saved viewer HTMLs as cards

The Python pipeline should read these templates, replace placeholders, and write the final HTML.

## Placeholders

| Placeholder | Source | Example |
|---|---|---|
| `{{ALBUM_NAME}}` | `data['album_name']` | Lanzarote Verde |
| `{{ALBUM_ID}}` | `data['album_id']` | 71d380f4-... |
| `{{IMMICH_URL}}` | `data['immich_url']` | https://your-immich-server.com |
| `{{TOTAL_ASSETS}}` | `data['total_assets']` | 273 |
| `{{COUNT}}` | `len(data['thumbnails'])` | 20 |
| `{{QUERY_TEXT}}` | User's original query | show me photos from... |
| `{{ICON_B64}}` | Plugin icon base64 | (12 KB PNG) |
| `{{HERO_SRC}}` | First thumbnail data URI | data:image/webp;base64,... |
| `{{IMAGES_JSON}}` | JS array of all thumbnails | [{id,src,date},...] |
| `{{PAGE_SIZE}}` | Photos per page | 6 |
| `{{ALBUMS_JSON}}` | JS array of related albums | [{id,name,total},...] |

## Required Features

### Header with Scroll Effects
- **🗾 icon** at 38px, shrinks to 26px when scrolled past 20px
- **Shadow** appears on scroll (light: subtle, dark: stronger)
- **Sticky** with backdrop-filter blur
- **Smooth transitions** (0.25s ease)

### View Modes (5)
1. **Detail** (▦) — 3-col grid, 4:3 ratio cards, hover lift, numbered badges
2. **Icon** (▣) — Compact square thumbnails, tight grid, no badges
3. **List** (☰) — Rows with 48px thumb, label, date, position number
4. **Masonry** (◫) — Variable-height tiles, 4 columns, Pinterest-style
5. **Gallery** (⛶) — Fullscreen overlay with single photo

### Themes (3)
- **Light** — Warm parchment (#f0ebe3 bg, #faf7f2 cards)
- **System** — Auto-detects via `prefers-color-scheme`, updates live
- **Dark** — Warm espresso (#1a1917 bg, #252420 cards)

All transitions use `0.3s cubic-bezier(0.4,0,0.2,1)`.
Accent color: #c8724e (terracotta) for light, #da7756 for dark.
Theme resolved in `<head>` before paint to prevent flash.

### Gallery Features
- Keyboard: ← → navigate, Space toggles slideshow, Esc closes
- Swipe: touch gestures for mobile/trackpad
- Slideshow: 4s interval, progress bar, play/pause toggle
- Keyboard hints: appear on open, fade after 3.5s
- Photo transition: opacity fade on change
- "Open in Immich →" link per photo

### Pagination
- IntersectionObserver infinite scroll (500px rootMargin)
- "Load more" button fallback
- Counter in header: "N / total"
- Hero shows "N of M loaded · T in album"

### Structure
- **Header**: plugin icon + title + counter + view pill + theme pill
- **Hero**: blurred first photo backdrop + album name + stats + query (compact, 140px height)
- **Grid**: paginated photo cards
- **Gallery overlay**: when activated
- **Related Albums**: section with "RELATED ALBUMS" header + polaroid cards (0, 1, or N)
- **Footer**: plugin icon + "Generated by" link

### Related Albums Section
- Rendered from `{{ALBUMS_JSON}}` array
- When array is empty, the entire section is hidden (`display:none`)
- When 1 album, single centered polaroid card
- When N albums, flex-wrapped row with 2rem gap
- Each polaroid uses a different rotation via `:nth-child` CSS (−2deg, +1.5deg, −3deg)
- Cover image: uses loaded thumbnails (album index maps to `ALL[i]`)
- Card links to `{IMMICH_URL}/albums/{album_id}`

### Cowork Actions Panel
A floating toolbar at the bottom of the viewer that bridges the HTML panel and Cowork chat:
- **Refine** — Copies a refinement prompt to clipboard for pasting in Cowork
- **Album** — Copies a "create album" command (uses selection if active)
- **Select** — Toggle selection mode: click photos to mark them (terracotta outline + checkmark)
- **Copy IDs** — Copies selected (or all) photo asset IDs to clipboard
- **Index** — Navigates back to `index.html` (the query dashboard)
- **Toast** — Confirmation messages appear above the panel

The panel slides up after the user scrolls 100px. This is the key interactivity
feature: users browse photos in the web panel, then paste commands into Cowork
to take action. The web and the chat work together.

### Index Dashboard (query-index.html)
A separate page listing all generated viewer HTMLs as cards (Option B):
- **Query cards** with mosaic cover (up to 3 thumbnails), query text, date, photo count
- **Action buttons** on hover: "Re-run" and "Create album" (copy to clipboard)
- **"New Query"** button copies a search prompt to clipboard
- **Stats** header showing total queries and photos
- Same header with 🗾 icon scroll effect
- Same theme system (light/system/dark)
- Sorted by date descending (newest first)

### Links
- Every photo card: `{IMMICH_URL}/photos/{asset_id}`
- Footer: `{IMMICH_URL}/albums/{album_id}`
- GitHub: `https://github.com/drolosoft/immich-photo-manager`
- Index button: `./index.html` (relative)

## Photo Data Format

Each entry in `{{IMAGES_JSON}}`:
```javascript
{
  id: "asset-uuid",       // For Immich link
  src: "data:image/...;base64,...",  // Embedded thumbnail
  date: "2024-03-15T..."  // ISO date from fileCreatedAt (may be empty)
}
```

When `date` is present, list view shows it formatted as "15 Mar 2024".
When empty, the date line is hidden gracefully.

## Album Data Format

Each entry in `{{ALBUMS_JSON}}`:
```javascript
{
  id: "album-uuid",     // For Immich album link
  name: "Album Name",   // Displayed on polaroid label
  total: 273            // Photo count shown below card
}
```

When `ALBUMS` is empty or undefined, the "Related Albums" section is hidden entirely.

## Index Dashboard Data Format

The index template (`assets/index-template.html`) uses a single placeholder:

`{{QUERIES_JSON}}` — JS array of all saved queries:
```javascript
{
  file: "lanzarote-verde.html",        // Viewer filename (relative link)
  query: "Show me photos from...",     // Original user query text
  album_name: "Lanzarote Verde",       // Album/search display name
  date: "2026-04-03T18:22:00",        // ISO date when generated
  count: 47,                           // Photos in this viewer
  total: 273,                          // Total in source album
  covers: ["data:image/...", ...]      // Up to 3 base64 thumbnails for card mosaic
}
```

## Pipeline Integration

### Generating a viewer
1. Read `assets/viewer-template.html`
2. Replace all `{{PLACEHOLDER}}` strings with actual data
3. Write to Documents folder as `{sanitized-name}.html`
4. Update the index (see below)
5. Share via `computer://` link

### Maintaining the index
Every time a new viewer is generated:
1. Read `assets/index-template.html`
2. Scan the output directory for existing `.html` files (excluding `index.html`)
3. For each viewer, extract metadata from embedded `<script>` data (ALBUM_NAME, ALL.length, etc.)
4. Build `QUERIES_JSON` array from all viewers
5. Replace `{{QUERIES_JSON}}` in the index template
6. Write/overwrite `index.html` in the same directory

This ensures the index always reflects the current set of saved queries.
Deleting a viewer `.html` file from Finder automatically removes it from the index
on the next generation.
